Search

Introduction to Object Oriented Programming | Scala | Spring 2020

Project Outline

I worked on search as a project for CS180 during Spring 2020, with a partner. The premise of this project was creating a search engine over a corpus of Wikipedia pages. The project was separated into two different components: an Indexer that would preprocess the information in the corpus and a Querier that would field a user query and then return the Wiki pages relevant to the query.

Indexer: Space and Time Efficiency

The Indexer was responsible for finding the frequency of each word (for term frequency score), the maximum frequency of each page (inverse document frequency score), and the rank of each page (for Google's page rank algorithm).

Querier: Interacting with the User

The second part of the project was the Querier, which processed the files that the Indexer wrote to and converted them back into data structures. And using those data structures to find each word associated with the user's query, computing their relevancy scores (term frequency x inverse document frequency) using the frequencies calculated by the Indexer and then returning the pages with the highest scores. And if the user had the "page rank" flag "activated" in their query, each page score would then have the page rank factored in.

A bug that I encountered was that initially I didn't account for the case when a user queried something like "dog breed alsdkjfalks". With that kind of query, I would simply return and say that "no results match your query", instead of just ignoring that word's score.
