DMalloc

Introduction to Computer Systems | C++ | Spring 2020

Project Overview

This project was centered around creating a debugger for the function Malloc, or when the user dynamically allocates memory from the heap. The main objectives for the project were to track information about dynamic memory allocation, catch programming errors like freeing the same block twice, or trying to allocate too much memory), and to detect memory overwrites.

Project Steps

Using Metadata

To store statistics such as the number of bytes the user requested it was suggested to store metadata before the user requested payload size, as well as some "padding" after the payload to check for boundary overwrites.

Debugger Steps

The bugs that we had to account for were integer overflow, invalid free and double frees, boundary overwrites, and freeing invalid pointers.

The most difficult part of this project was understanding what to do conceputally and using a data structure appropriately.

The most difficult bug to track was tracking invalid frees, for example trying to free a pointer that was actually allocated in another block. In general writing kfree(), there was a lot of checks — there needed to be a way to guarantee that I was actually accessing the metadata pointer, a way to track if that pointer pointed to an address that was allocated data on the heap, and a way to track if that pointer had already been freed. All of these checks were in place to make sure that in situations where users would copy data from other allocated regions that would overwrite the existing data — in other words, copying and overwriting existing metadata.
